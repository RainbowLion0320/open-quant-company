# Support

Astrolabe is open source research infrastructure. Support is best handled through public, reproducible GitHub issues.

## Where to Ask

- Use bug reports for reproducible defects.
- Use feature requests for product or architecture proposals.
- Use documentation issues for unclear, outdated, or missing docs.
- Use questions for setup or usage problems that are not confirmed bugs.

Before opening an issue, search existing issues and check the README, `docs/`, and `wiki/`.

## What to Include

Useful support requests include:

- The command, Web route, or workflow you were using.
- Expected behavior and actual behavior.
- Relevant logs, tracebacks, or screenshots.
- OS, Python version, Node version, and browser version when relevant.
- Whether secrets or provider credentials were configured.

Remove API tokens, webhook URLs, account IDs, private trading records, and local data before posting.

## Boundaries

The project does not provide:

- Investment advice.
- Trading recommendations.
- Return guarantees.
- Support for private provider accounts or proprietary datasets.
- Debugging for untracked local changes without a reproducible example.
