# Code of Conduct

## Our Standard

Astrolabe is a technical open source project for quant research infrastructure. The project welcomes contributors who communicate clearly, argue from evidence, and keep discussions focused on improving the software.

Expected behavior:

- Be respectful and professional in issues, pull requests, reviews, and discussions.
- Explain technical disagreements with concrete evidence, reproducible examples, or references to project goals.
- Assume contributors have different backgrounds in software engineering, finance, data engineering, and research.
- Keep security-sensitive information, API keys, private datasets, and personal trading information out of public threads.
- Make it clear when a statement is opinion, research hypothesis, or verified behavior.

Unacceptable behavior:

- Harassment, insults, threats, discrimination, or repeated personal attacks.
- Publishing another person's private information without explicit permission.
- Posting secrets, credentials, private trading records, or proprietary datasets.
- Presenting financial advice, guaranteed returns, or investment recommendations as project guidance.
- Deliberately derailing technical discussions or repeatedly ignoring maintainer moderation.

## Scope

This Code of Conduct applies to all project spaces, including GitHub issues, pull requests, discussions, documentation reviews, and community channels that are explicitly associated with this repository.

## Enforcement

Maintainers may remove comments, close issues, block contributors, or reject contributions that violate this Code of Conduct. Enforcement decisions should be proportional, documented when appropriate, and focused on protecting the project and its contributors.

If you need to report conduct concerns, contact the repository owner through GitHub profile contact channels. Do not post private or sensitive reports in public issues.
